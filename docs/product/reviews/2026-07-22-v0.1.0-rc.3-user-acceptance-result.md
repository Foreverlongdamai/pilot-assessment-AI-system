## Observed acceptance defects

Independent use of the exact RC.3 package found that the product still did not provide the intended
continuous expert-editing experience:

1. the user could only commit all staged model changes through the close workflow; there was no
   active **Save All** command that kept the software open;
2. a normal graph node followed the pointer during drag but returned to its original position on
   release;
3. the five green Raw Input Family projection roots could not be dragged.

The RC.3 Assessment-purpose execution, packaged icon, global node archive/delete action and portable
root-layout corrections remain valid engineering improvements. They do not make the exact RC.3 ZIP
acceptable when the editing defects above remain.

## Required correction

The replacement candidate must include D-088 and D-089. **Save All** and `Ctrl+S` must flush all
open editors and pending layout changes, atomically commit the software-owned edit session and keep
the application open. Normal nodes and the five display-only Raw Input Family roots must stay at the
released pointer position, persist through the edit session and survive Save All/restart. Raw family
roots remain layout-only projections and must not enter Evidence/BN/CPT or RunSnapshot semantics.

RC.3 remains immutable. Its tag, ZIP identity, checksum and verification report continue to describe
the exact package that was tested. The local `dist/releases` cache may be cleaned as an operational
convenience, but Git tags and historical review records are not deleted or reassigned.
