## 1. 验证范围与边界

本记录验证 RC.2 用户验收提出的四项返修：Assessment 用途的技术执行、发布窗口图标、全局删除节点，以及按住指针拖动节点。它同时重新关闭 source、C#、文档、tagged build、ZIP、仓库外 restricted-PATH、可编辑源码、operator 扩展、轻量 Assessment run 和桌面启动门。

本记录不表示用户已经接受实际使用体验，也不证明 starter Evidence、operator 参数、阈值、BN 拓扑、CPT 或能力后验具有科学有效性。Assessment 在技术 ready 时可以运行，但仍明确携带 `run.assessment_not_authorized` warning 与 `formal_run_authorized=false`。

## 2. 修订结果

- Assessment purpose 不再因科学授权状态被技术阻断；仓库外 run 实测 `purpose=assessment`、`run_state=completed`、`authorization_warning=true`、`formal_run_authorized=false`。
- `AppWindow.SetIcon` 使用 executable-relative absolute path；`Assets/AppIcon.ico` 进入 build/publish/package。最终候选的标题栏显示 eVTOL 图标。
- Model Studio 工具栏与节点上下文菜单提供全局删除；确认后，Python 后端在单一 edit-session 事务中从所有受影响 TaskScheme 停用节点及依赖闭包，再归档全局节点。历史 RunSnapshot 不变。
- 图节点通过 handled pointer events 捕获按住的左键指针；移动超过 4 px 即进入拖动，不再依赖 350 ms 驻留计时器。最终候选实测 `ECG.channels` 跟随指针移动，随后 `Ctrl+Z` 恢复原位。
- 可见复测额外发现并关闭 WinUI runtime language toggle crash；语言切换只更新显式 resource context 和 UI culture，不再写进程级 `ApplicationLanguages.PrimaryLanguageOverride`。
